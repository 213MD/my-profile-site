Custom Output Style 생성 방법이 파일 기반 방식으로 변경되었습니다!
기존 /output-style:new 명령어 대신, 이제는 Markdown 파일을 직접 작성하여 더 강력하고 체계적인 커스터마이징이 가능합니다.

👉 나만의 출력 스타일 만들기 강의교안

🎯 미션 목표
클로드 코드의 출력 스타일(Output Style)을 내 수준과 학습 목적에 맞게 커스터마이징하고, 반복적인 개선을 통해 최적화된 나만의 스타일을 완성합니다.

📋 미션 내용
1단계: 출력 스타일의 중요성 이해하기
💡 왜 출력 스타일이 중요한가요?
클로드 코드는 초보자부터 전문가까지 모두를 위한 도구입니다. 하지만 사람마다 필요한 설명의 깊이가 다릅니다:

초보자: 코드 한 줄 한 줄에 대한 자세한 설명과 학습 가이드 필요
중급자: 핵심 로직과 주요 개념 위주로 설명, 패턴과 인사이트 제공
고급자: 간결한 요약과 중요한 결정 사항만, 효율적인 작업 중심
출력 스타일을 설정하면 클로드가 내 수준에 맞는 방식으로 대답합니다!

💡 내장된 출력 스타일 3가지:
Default: 효율적인 소프트웨어 개발 작업에 최적화 (기본값)
Explanatory: 코드 작성 중간중간 교육적인 "인사이트"를 제공하여 코드베이스 패턴과 구현 선택을 이해할 수 있도록 도움
Learning: 협업형 학습 모드로, 클로드가 작은 부분을 TODO(human) 마커로 표시하여 직접 구현해보도록 유도
2단계: 클로드 코드 실행 및 현재 출력 스타일 확인하기
간단한 테스트 프로젝트 폴더를 만듭니다:

cd ~/workspace
mkdir output-style-test
cd output-style-test
claude
클로드 코드에서 /output-style 명령어를 입력하여 출력 스타일 메뉴를 엽니다:

/output-style
현재 어떤 출력 스타일이 설정되어 있는지, 어떤 내장 스타일들이 있는지 확인합니다.

3단계: 내장 출력 스타일 테스트해보기
먼저 내장된 Explanatory 스타일을 시험해봅니다:

/output-style explanatory
이 스타일로 간단한 질문을 해봅니다:

간단한 투두 리스트 HTML을 만들어줘.
클로드가 코드 작성 중간에 "인사이트"를 제공하는 것을 확인합니다.

다음으로 Learning 스타일을 시험해봅니다:

/output-style learning
같은 질문을 다시 해봅니다:

간단한 계산기 HTML을 만들어줘.
클로드가 TODO(human) 마커를 남기며 직접 구현하도록 유도하는 것을 확인합니다.

💡 각 스타일의 차이를 느껴보세요: 어떤 스타일이 내 학습 목적에 맞는지 파악하는 것이 중요합니다!

4단계: 나만의 커스텀 출력 스타일 파일 만들기
이제 나만의 출력 스타일을 만들어봅니다. Markdown 파일을 직접 작성하는 방식입니다.

📁 파일 생성 위치
모든 프로젝트에서 사용 (권장):

~\.claude\output-styles\beginner.md
